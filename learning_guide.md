# InstaClone 코드 학습 가이드

이 문서는 InstaClone 프로젝트의 소스 코드를 학습하기 위한 순서와 가이드입니다.
웹 개발의 흐름(데이터 -> 서버 -> 클라이언트)을 따라가는 것이 가장 이해하기 쉽습니다.

## 1단계: 데이터와 백엔드 (서버의 기초)

가장 먼저 "어떤 데이터를 다루는가?"를 정의하고, 그 데이터를 "어떻게 주고받을 것인가?"를 이해해야 합니다.

### 1-1. `backend/models.py` (데이터 설계도)

- **학습 목표**: 데이터베이스 테이블(Member, Post, Comment 등)이 어떻게 정의되는지 이해합니다.
- **핵심 개념**:
  - `db.Model`: 데이터베이스 테이블과 매핑되는 클래스
  - `db.Column`: 테이블의 컬럼(열) 정의
  - `relationship`, `ForeignKey`: 테이블 간의 관계 (예: 누가 쓴 글인가?)

### 1-2. `backend/app.py` (API 서버 로직)

- **학습 목표**: 프론트엔드에서 보낸 요청을 어떻게 처리하고 응답하는지 흐름을 파악합니다.
- **핵심 개념**:
  - `@app.route`: URL 주소와 파이썬 함수 연결
  - `request`, `jsonify`: 요청 데이터 받기 및 JSON 응답 보내기
  - `db.session`: 데이터베이스 저장/수정/삭제 트랜잭션 관리

---

## 2단계: 프론트엔드 구조 (화면의 기초)

서버가 준비되었으면, 리액트 앱이 어떻게 시작되고 구성되는지 봅니다.

### 2-1. `frontend/src/main.jsx` (진입점)

- **학습 목표**: 리액트 앱이 HTML(`index.html`)에 어떻게 부착되는지 봅니다.
- **핵심 개념**: `createRoot`, `render`

### 2-2. `frontend/src/App.jsx` (라우팅 및 전역 상태)

- **학습 목표**: 페이지 이동(라우팅)은 어떻게 처리하며, 로그인 정보 같은 전역 데이터는 어떻게 관리하는지 배웁니다.
- **핵심 개념**:
  - `Routes`, `Route`: URL에 따라 보여줄 컴포넌트 결정
  - `useState`: 현재 로그인한 사용자 정보 저장
  - `useEffect`: 앱 실행 시 로그인 유지(세션 확인) 로직

---

## 3단계: 주요 기능 구현 (인증과 피드)

### 3-1. 인증 페이지 (`Login.jsx`, `Register.jsx`)

- **학습 목표**: 사용자의 입력을 받아 서버로 보내는 기본적인 폼(Form) 처리를 배웁니다.
- **핵심 개념**: `onChange` (입력값 제어), `axios.post` (서버 요청)

### 3-2. `frontend/src/pages/Feed.jsx` (메인 화면)

- **학습 목표**: 서버에서 데이터를 가져와서(`GET`) 화면에 뿌려주는 목록 렌더링을 배웁니다.
- **핵심 개념**: `useEffect`를 이용한 데이터 로딩, `map` 함수를 이용한 목록 출력

### 3-3. `frontend/src/components/PostCard.jsx` (핵심 컴포넌트)

- **학습 목표**: 하나의 컴포넌트 안에서 다양한 상호작용(좋아요, 댓글, 삭제)이 어떻게 일어나는지 학습합니다. **가장 공부할 내용이 많은 파일입니다.**
- **핵심 개념**:
  - 복잡한 상태 관리 (좋아요 수, 댓글 목록)
  - 조건부 렌더링 (내 글일 때만 삭제 버튼 보이기)
  - 대댓글 로직 (상태를 이용한 UI 변화)

---

## 4단계: 유틸리티 및 심화

### 4-1. `frontend/src/components/Header.jsx`

- **학습 목표**: 페이지 상단 네비게이션과 로그아웃 처리를 봅니다.

### 4-2. `frontend/src/components/CreatePostModal.jsx`

- **학습 목표**: 모달 창(팝업) 구현과 파일(이미지) 업로드 방식을 배웁니다.
- **핵심 개념**: `FormData` 객체 사용법 (파일 전송 시 필수)

### 4-3. `frontend/src/pages/Profile.jsx`

- **학습 목표**: URL 파라미터(`:userID`)를 피드에 활용하여 특정 유저의 글만 필터링하는 법을 배웁니다.
